every time you run it,
    * checks process node version
    * if node version below 10.10, prints out notice to run with 10.10+ and exits
    
then, it goes into a CLI loop. the menu options differ depending on state.
if the tool directory does not exist,
we show
i) init   (set up the tool directory and generate a default config file. Optionally adds ./i18n-janitor/results to .gitignore)

after init runs, or if the tool directory already exists on startup, the options are
f) find unused tokens (uses config file to find unused tokens and saves results to ./i18n-janitor/results/unused-tokens-[timestamp].json
p) purge unused tokens (uses the results file to remove unused tokens from all locale files. re-generates results file)
q) quit
